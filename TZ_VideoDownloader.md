## 1. Цель и контекст

### 1.1. Цель проекта
Создать Chrome-расширение, которое автоматически обнаруживает видео на текущей странице и позволяет скачать их в один клик — в том числе с платформ, требующих авторизации (GetCourse, VK, и др.).

### 1.2. Контекст
Обучающие платформы (GetCourse и аналоги) не предоставляют прямой кнопки скачивания видео. Видео доставляется через защищённые HLS-потоки (`.m3u8`) с токенами в URL. Стандартные инструменты (yt-dlp из терминала) требуют ручной передачи cookies и не интегрированы в браузер.

**Ключевое преимущество расширения:** оно работает внутри браузера, где уже есть активная сессия пользователя, и может передать cookies и перехваченные URL на бэкенд — без ручного копирования.

### 1.3. Out of scope (первый этап)
- iOS и Safari
- Мультипользовательский режим
- Скачивание субтитров, глав, аудио отдельно
- Скачивание плейлистов целиком

## 3. Архитектура

### 3.1. Обзор

Система состоит из двух частей: **Chrome Extension** (клиент) и **Download Backend** (сервер на VPS). Такое разделение выбрано по трём причинам:

1. HLS-поток состоит из сотен сегментов — мержить их в браузере нереально, нужен ffmpeg
2. Скачивание тяжёлых файлов через браузер ненадёжно (таймауты, потеря вкладки)
3. На VPS можно ставить очередь задач и хранить историю

```
┌─────────────────────────────────┐
│         CHROME BROWSER          │
│                                 │
│  ┌──────────┐  ┌─────────────┐  │
│  │  Content │  │  Background │  │
│  │  Script  │→ │Service Worker│ │
│  │(наблюдает│  │(перехват    │  │
│  │ за DOM)  │  │ запросов)   │  │
│  └──────────┘  └──────┬──────┘  │
│                       │         │
│              ┌────────▼──────┐  │
│              │  Popup UI     │  │
│              │ (список видео,│  │
│              │  прогресс)    │  │
│              └────────┬──────┘  │
└───────────────────────┼─────────┘
                        │ HTTPS API
                        │ + cookies
              ┌─────────▼──────────┐
              │   VPS BACKEND      │
              │                    │
              │  FastAPI           │
              │  + yt-dlp          │
              │  + ffmpeg          │
              │  + Task Queue      │
              │    (Redis/asyncio) │
              └────────────────────┘
```

### 3.2. Компоненты расширения

| Компонент | Файл | Ответственность |
|-----------|------|-----------------|
| Background Service Worker | `background.js` | Перехват сетевых запросов (`.m3u8`, `.mp4`, iframe src), хранение найденных URL |
| Content Script | `content.js` | Сканирование DOM на `<video>`, `<iframe>`, data-атрибуты; MutationObserver для SPA |
| Popup | `popup.html/js` | UI: список видео, кнопка скачать, прогресс, история |
| Options | `options.html` | Настройка URL бэкенда, API-ключ |

### 3.3. Компоненты бэкенда

| Компонент | Технология | Ответственность |
|-----------|------------|-----------------|
| API сервер | FastAPI | Приём задач, отдача статуса и файлов |
| Загрузчик | yt-dlp | Скачивание с YouTube, VK, Vimeo, GetCourse |
| Обработчик | ffmpeg | Мерж HLS-сегментов, конвертация в mp4 |
| Очередь задач | asyncio + in-memory | Управление параллельными загрузками |
| Хранилище | Локальная FS (`/tmp/videograb/`) | Временное хранение скачанных файлов |

## 5. Детали реализации

### 5.1. Как расширение находит видео

GetCourse и похожие платформы используют несколько способов встраивания видео:

```
Способ 1: Прямой <video> тег
→ Content Script находит <video src="..."> или <source src="...">

Способ 2: HLS через JavaScript (самый частый на GetCourse)
→ Background SW перехватывает запросы к *.m3u8
→ Сохраняет URL + заголовки запроса (включая cookies/auth-headers)

Способ 3: Вложенный iframe (Kinescope, Vimeo, YouTube)
→ Content Script извлекает src из <iframe>
→ Background SW дополнительно слушает запросы внутри iframe
```

### 5.2. Передача авторизации на бэкенд

Это ключевая техническая задача. Решение:

```javascript
// background.js — сбор cookies для домена
async function getCookiesForUrl(url) {
  const cookies = await chrome.cookies.getAll({ url });
  // Преобразуем в строку формата "name=value; name2=value2"
  return cookies.map(c => `${c.name}=${c.value}`).join('; ');
}

// При отправке задачи на бэкенд
const payload = {
  video_url: detectedUrl,
  cookies: await getCookiesForUrl(tabUrl),
  referer: tabUrl,
  user_agent: navigator.userAgent
};
```

На бэкенде yt-dlp принимает cookies как строку:
```python
ydl_opts = {
    'http_headers': {
        'Cookie': task.cookies,
        'Referer': task.referer,
        'User-Agent': task.user_agent,
    }
}
```

### 5.3. Безопасность бэкенда

Бэкенд принимает задачи только с API-ключом. Ключ хранится в расширении в `chrome.storage.local` (не в коде).

```
Extension → POST /api/download
Headers: X-API-Key: <secret_key>
Body: { video_url, cookies, referer, user_agent, format }
```

### 5.4. Поток скачивания

```
1. Пользователь нажимает "Скачать" в popup
2. Popup → background.js → POST /api/download на VPS
3. Бэкенд создаёт задачу, возвращает task_id
4. Popup polling: GET /api/status/{task_id} каждые 2 сек
5. Статусы: queued → downloading → processing → ready → error
6. При статусе ready — появляется кнопка "Сохранить файл"
7. Кнопка открывает GET /api/file/{task_id} → браузер скачивает mp4
8. Файл на VPS удаляется через 1 час
```

## 12. ADR — Ключевые архитектурные решения

### ADR-001: Chrome Extension, а не Desktop App

**Решение:** Chrome Extension (MV3)

**Почему:** Расширение имеет прямой доступ к cookies браузера без каких-либо дополнительных манипуляций. Пользователь уже залогинен — расширение просто передаёт эту сессию на бэкенд. Desktop-приложение потребовало бы отдельного механизма авторизации на каждой платформе. Кроме того, одна кодовая база работает и на Mac, и на Windows.

**Компромисс:** Ограничения MV3 (нет persistent background), требует установки из Chrome Web Store или в режиме разработчика.

---

### ADR-002: Бэкенд на VPS, а не всё локально

**Решение:** FastAPI на VPS для обработки загрузок

**Почему:** HLS-поток — это манифест + сотни `.ts`-сегментов, которые нужно скачать и смержить через ffmpeg. Делать это в браузере невозможно. VPS уже есть, ffmpeg + yt-dlp там работают стабильно. Дополнительный бонус — можно скачивать в фоне, закрыв вкладку.

**Компромисс:** Нужен HTTPS на VPS, файлы временно хранятся на сервере.

---

### ADR-003: yt-dlp как основной загрузчик

**Решение:** yt-dlp вместо самописного загрузчика

**Почему:** yt-dlp поддерживает 1000+ платформ, умеет работать с cookies, автоматически выбирает лучшее качество, регулярно обновляется. Написать аналог с нуля — месяцы работы.

**Компромисс:** Для GetCourse (Kinescope) может потребоваться прямой URL из перехваченного запроса, т.к. yt-dlp не всегда умеет парсить страницу GetCourse. Стратегия: расширение перехватывает m3u8 URL и передаёт его напрямую в yt-dlp — это обходит любую защиту страницы.

## Приложения

### A. Необходимые разрешения расширения (manifest.json)
```json
{
  "permissions": [
    "webRequest",
    "cookies",
    "storage",
    "notifications",
    "activeTab"
  ],
  "host_permissions": [
    "<all_urls>"
  ]
}
```

> `<all_urls>` нужен для перехвата запросов на любом домене и чтения cookies. При публикации в Web Store это потребует объяснения — для личного использования устанавливается в режиме разработчика.

### B. Глоссарий

| Термин | Определение |
|--------|-------------|
| HLS | HTTP Live Streaming — протокол доставки видео сегментами через `.m3u8` плейлист |
| MV3 | Manifest V3 — текущий стандарт Chrome Extensions с ограниченным Service Worker |
| yt-dlp | Fork youtube-dl, инструмент скачивания видео с 1000+ платформ |
| Kinescope | Российский видеохостинг, используемый GetCourse |
| ffmpeg | Утилита для конвертации и обработки медиафайлов |
