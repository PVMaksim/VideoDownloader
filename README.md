# VideoGrab

Chrome-расширение для скачивания видео с GetCourse, YouTube, VK и Instagram. Работает с платформами требующими авторизации — использует cookies активной сессии браузера.

## Как это работает

```
Chrome Extension → перехватывает видео-поток → передаёт на VPS
VPS (yt-dlp + ffmpeg) → собирает HLS сегменты → отдаёт mp4
```

Расширение автоматически находит видео на странице. Нажимаешь "Скачать" — файл скачивается через сервер и сохраняется на компьютер.

## Поддерживаемые платформы

| Платформа | Статус | Авторизация |
|-----------|--------|-------------|
| GetCourse (Kinescope) | ✅ Работает | Автоматически (cookies) |
| YouTube | ✅ Работает | Автоматически (cookies) |
| VK Video | 🧪 Тестируется | Автоматически (cookies) |
| Instagram Reels/Posts | ⚠️ Нестабильно | Автоматически (cookies) |

## Установка расширения

1. Скачай архив `videograb-extension.zip` и распакуй
2. Открой `chrome://extensions/`
3. Включи **Режим разработчика** (правый верхний угол)
4. Нажми **Загрузить распакованное** → выбери папку `videograb-extension`
5. Открой настройки расширения (⚙️) → укажи URL бэкенда и API ключ

## Деплой бэкенда на VPS

### Требования
- Ubuntu 22.04+
- Docker + docker-compose
- Nginx + certbot (для HTTPS)
- 10+ ГБ свободного места

### Быстрый старт

```bash
# Скопируй папку backend/ на VPS
scp -r backend/ user@your-vps:/home/deploy/videograb/

# Зайди на VPS
ssh user@your-vps
cd /home/deploy/videograb

# Создай .env из примера
cp .env.example .env
nano .env  # задай API_KEY и остальные параметры

# Запусти
docker-compose up -d

# Проверь
curl http://localhost:8000/api/health
```

### Nginx конфиг (HTTPS)

```nginx
server {
    server_name api.yourdomain.com;

    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_read_timeout 300s;  # важно для длинных загрузок
    }

    listen 443 ssl;
    ssl_certificate /etc/letsencrypt/live/api.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.yourdomain.com/privkey.pem;
}
```

```bash
# Получить SSL сертификат
certbot --nginx -d api.yourdomain.com
```

## Переменные окружения

Все переменные описаны в `backend/.env.example`:

```env
API_KEY=          # Секретный ключ — придумай сложный, вставь в настройки расширения
DOWNLOAD_DIR=     # Папка для временных файлов (по умолчанию /tmp/videograb)
FILE_TTL_HOURS=   # Через сколько часов удалять скачанные файлы (по умолчанию 2)
MAX_CONCURRENT=   # Максимум параллельных загрузок (по умолчанию 3)
```

## API бэкенда

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/health` | Проверка работоспособности |
| POST | `/api/download` | Создать задачу скачивания |
| GET | `/api/status/{task_id}` | Статус задачи (0–100%) |
| GET | `/api/file/{task_id}` | Скачать готовый mp4 |
| DELETE | `/api/task/{task_id}` | Удалить задачу и файл |

Все запросы требуют заголовок `X-API-Key: <твой ключ>`.

Главный вопрос — бизнес-модель
Есть два принципиально разных пути:

Путь А — B2C инструмент для пользователей (как yt-dlp с UI) Пользователь платит за удобство скачивания своего контента. Монетизация: подписка $5–15/мес.

Путь Б — B2B инструмент для школ и авторов Продаёшь школам возможность давать студентам офлайн-доступ к урокам. Монетизация: корпоративные лицензии.

Путь А реалистичнее на старте. Путь Б — больше денег, но длинный цикл продаж.

Что нужно доработать технически
1. Мультипользовательский бэкенд
Сейчас бэкенд — твой личный сервер без авторизации пользователей. Нужно:

Текущее:          Нужно:
1 API ключ        User accounts (email + password)
1 сервер          Очередь задач (Celery + Redis)
in-memory tasks   База данных (PostgreSQL)
нет лимитов       Лимиты по тарифу (X скачиваний/мес)
2. Платёжная система
Stripe или ЮKassa (если РФ аудитория). Интеграция несложная — 2–3 дня работы. Модель: Free (3 скачивания/день) → Pro ($9/мес, безлимит).

3. Личный кабинет
Простой веб-сайт: регистрация, оплата, история скачиваний, управление подпиской. Можно на Next.js или даже на no-code (Webflow + кастомный бэкенд).

4. Расширение в Chrome Web Store
Сейчас расширение ставится только в режиме разработчика. Для публикации нужно:

Пройти ревью Google (1–2 недели)
Заплатить $5 (разовый взнос разработчика)
Убрать <all_urls> или обосновать его использование
Это открывает доступ к миллионам пользователей
5. Стабильность и мониторинг
Sentry для отслеживания ошибок
Очередь задач чтобы сервер не падал под нагрузкой
Автообновление yt-dlp (платформы часто меняют защиту)
Юридические риски — важно понимать
Это самая серьёзная часть. Честно:

Серая зона. Скачивание контента для личного использования в большинстве стран легально. Но перепродажа инструмента для скачивания чужого контента — это риск претензий от платформ.

GetCourse скорее всего закроет глаза — их пользователи скачивают для офлайн-просмотра, не для пиратства. Это скорее UX-проблема платформы.

YouTube — нарушение ToS, но YouTube не преследует частных пользователей. Проблема если масштаб станет заметным.

Что снижает риски:

Позиционирование "для скачивания своего контента" (курсы которые ты купил)
Не помогать скачивать контент других людей (watermark detection, например)
Terms of Service со стороны твоего сервиса
